Web Development - Tutorial

Idea

This project is going to be the framework for a web-development-tutorial.

The website currently has just a few pages about HTML and CSS, but it is build to be easily extended.
The header as well as the sidebar in the html section already hold dummy pages to show the potential of this guide.

Speaking of those elements, the user can access different parts of the tutorial via the header or the sidebar within a language.


Development & Troubleshooting

The following paragraphs are a summary of all the milestones of this project.


index.html

Every webpage starts with the index.html.
Albeit the section you're currently reading was added bit by bit later on in the development.


Header & sidenav

The header was up next being the first big element on the website.
The biggest problem creating it, was the arrangement of the elements (icons & texts) and the text-align and vertical-align within them. Building the sidenav was very similar.

A small issue was with position: fixed of both elements. First working with position: sticky lead to a few overlaps and ugly behaviours.


Content

Due to wrong/bad css the header was always just as big as the biggest child it contained, which caused ugly effects later on.
Another funny bug was content scrolling over the fixed header. This was fixed with a z-index.


Dummy Pages

Dummy Pages were added to give the illusion of a much more advanced website.
I made the mistake of copy pasting the first build. When I wanted to change something a few hours later, I had to copy all of the content again.


Infopanel

Stretching the website horizontally made it look a bit empty, so I added an infopanel on the right.
I used a lot of code from the sidenav but that needed a lot of refactoring. The panel holds information about social links and another project of mine for 'Informationsvisualisierung'.


Responsive Design

At this point I already created a Github repository for the code to be on the safe side and get a version control too.
So what I did was to open the website on my phone just to realize that it was a complete mess.
I made the header, sidenav and the infopanel responsive, as well as some images on the sub pages.


Authors Page

The authors page was the last of the three main pages to implement. Because of the requirements I had to build a complex table.
It took quite the time to get the formatting right, because of weird arrangements of the table and its cells.
But at the end I got a nice, non smartphone-friendly page to praise myself.


Future Idea

When the sidenav is on top of the page when visiting with a smartphone, it should not be completly visible right from the start.
Right now it isn't that big of a problem, but when there are more sub pages per language, an expandable menu is necessary. Otherwise the user would have to scroll over all navigation points first.

Another thing to fix is the responsive design of the header. An idea would be to put them on top of each other or put the languages inside a menu button.
